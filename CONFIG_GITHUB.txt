📌 AUTHENTIFICATION GITHUB
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pour créer un Personal Access Token:

1. Aller sur: https://github.com/settings/tokens
2. Cliquer: "Generate new token" → "Tokens (classic)"
3. Nom: ElAmira ERP
4. Expiration: No expiration (ou 1 an)
5. Permissions: Cocher "repo" (tous les sous-items)
6. Générer et COPIER le token
7. Utiliser ce token comme mot de passe lors du push


📌 RÉSOLUTION DE PROBLÈMES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problème: "remote origin already exists"
Solution:
    git remote remove origin
    git remote add origin https://github.com/Amine08kimo/elamira-erp.git

Problème: "failed to push"
Solution:
    git pull origin main --rebase
    git push origin main

Problème: Authentification échoue
Solution:
    → Utiliser un Personal Access Token
    → Ou installer GitHub Desktop
