PROBLÈME : Les scripts .bat ne fonctionnent pas dans PowerShell

SOLUTION : Appliquer manuellement les changements


═══════════════════════════════════════════════════════════════
  ÉTAPE 1 : Nettoyer la Base de Données
═══════════════════════════════════════════════════════════════

1. Aller dans le dossier : database\

2. Supprimer le fichier : odoo_clone_dz.db
   (Clic droit → Supprimer)


═══════════════════════════════════════════════════════════════
  ÉTAPE 2 : Appliquer le Design Équilibré  
═══════════════════════════════════════════════════════════════

1. Aller dans : core\assets\themes\

2. Copier le fichier : odoo_theme_balanced.qss

3. Renommer la copie en : odoo_theme.qss
   (Remplacer l'ancien si demandé)


═══════════════════════════════════════════════════════════════
  ÉTAPE 3 : Appliquer Dashboard Équilibré
═══════════════════════════════════════════════════════════════

1. Aller dans : modules\dashboard\

2. Copier le fichier : views_balanced.py

3. Renommer la copie en : views.py
   (Remplacer l'ancien si demandé)


═══════════════════════════════════════════════════════════════
  ÉTAPE 4 : Vérifier Module Maintenance
═══════════════════════════════════════════════════════════════

1. Vérifier que le dossier existe : modules\maintenance\

2. Le dossier doit contenir :
   ✅ __init__.py
   ✅ maintenance_module.py
   ✅ models.py
   ✅ controller.py
   ✅ views.py

3. Si le dossier manque, il a été créé mais pas sauvegardé


═══════════════════════════════════════════════════════════════
  ÉTAPE 5 : Lancer l'Application
═══════════════════════════════════════════════════════════════

Dans PowerShell :

    cd "c:\Users\ms\Desktop\Application POS ALAMIRA@COM\ElAmiraVer0.01"
    
    python main.py

  SI PROBLÈME : Base de Données Pleine
═══════════════════════════════════════════════════════════════

ERREUR : "database or disk is full"

SOLUTION :

1. Vérifier espace disque C:
   - Doit avoir minimum 1 GB libre

2. Nettoyer :
   - Corbeille Windows
   - Fichiers temporaires (%TEMP%)
   - Téléchargements

3. Ou déplacer application sur D: ou E:


═══════════════════════════════════════════════════════════════
  ALTERNATIVE : Utiliser PowerShell Script
═══════════════════════════════════════════════════════════════

Dans PowerShell (Administrateur) :

    cd "c:\Users\ms\Desktop\Application POS ALAMIRA@COM\ElAmiraVer0.01"
    
    .\MISE_A_JOUR_RAPIDE.ps1

Si erreur "scripts désactivés" :

    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
    
    .\MISE_A_JOUR_RAPIDE.ps1
